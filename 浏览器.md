# 事件循环机制

1、执行当前 同步代码
2、执行所有 微任务（直到队列为空）
3、尝试 UI 渲染
4、取出 一个宏任务 执行
5、重复循环


# 管线

1、构建 DOM 树
  解析 HTML，生成 DOM 树（文档对象模型）
2、构建 CSSOM 树
  解析 CSS，生成 CSSOM 树（样式对象模型）
3、生成 渲染树（Render Tree）
  把 DOM + CSSOM 结合，只保留可见节点→ 看不见的（display:none、head）不会进渲染树
4、布局（Layout / Reflow）
  计算每个元素的几何位置：宽高、坐标
5、绘制（Paint）
  填充像素：颜色、图片、阴影、边框等
6、合成（Composite）
  把多个图层合成为一帧，最终显示到屏幕

## 阻塞问题（面试必问）
HTML 阻塞渲染吗？不阻塞，边下载边解析。
CSS 阻塞渲染吗？阻塞渲染，CSSOM 没构建完，页面不渲染。
JS 阻塞渲染吗？阻塞 HTML 解析 + 阻塞渲染因为 JS 可能修改 DOM/CSS，浏览器必须等 JS 执行完。
怎么优化？
CSS 放 head，尽早加载
JS 放 body 底部，或加 async/defer
用 transform/opacity 做动画，走合成线程

## 合成（Composite） 
是浏览器渲染管线最后一步。
它把页面拆成多个独立图层，交给 GPU 加速。
合成线程独立于主线程，不阻塞、不卡顿。
只有 transform、opacity 会直接走合成。
过程：计算图层变换 → GPU 叠加混合 → 输出一帧画面到屏幕。
性能开销：重排 > 重绘 > 合成。
