# LCP 的最大内容取决于什么
1、元素：图片、video、css 背景图、块级元素如 div
2、最大判断规则
  可视区内，超出忽略
  尺寸计算：渲染尺寸和原始尺寸最小值（防止小图被拉伸）
  动态更新规则：有更大的元素出现后替换
  权重修正：视口中心区域权重越高，边缘越远
3、影响时间
  服务器响应：html 首字节
  关键资源加载：
    LCP 图片和字体的耗时
    阻塞渲染的 css/js延迟资源发现与渲染
  客户端渲染与主线程
    js长任务阻塞主线程，延迟 LCP 绘制 
    字体未加载导致文本渲染延迟
  资源优先级与未加载
    未用 fetchpriority="high"、preload 提升 LCP 资源优先级，导致加载靠后。

# 有一个计算量特别大的函数影响性能，用什么办法优化？怎么给长任务分片？
1、算法优化：通过缓存、逻辑优化等，减少计算工作量
2、通过 Web Worker 方式移除主进程
3、长任务切片：通过 requestIdleCallback、requestAnimationFrame 分批执行，每批后释放主线程；
4、react 场景：通过 useTransition、useDeferredValue 等 避免阻塞交互

# 重排和重绘
重排：元素位置或尺寸变化，浏览器需要重新计算布局，称为重排。
重绘：元素颜色、背景、透明度等不影响布局的样式变化，浏览器需要重新绘制元素，称为重绘。
## 如何避免重排
1、批量操作：避免频繁操作 DOM，将多个操作合并为一次。
2、使用离线 DOM：将 DOM 元素离线操作，操作完成后再添加到文档中，避免多次重排。
3、使用 CSS 动画：将动画效果应用于 CSS 属性，避免直接操作 DOM 元素。
4、使用 transform 和 opacity 等属性：只触发合成层，不触发重排重绘。
5、避免频繁读取布局属性（offsetTop、scrollWidth 等），读取会强制刷新布局队列
6、开启 will-change 提示浏览器提前优化。
7、避免 table 布局，微小改动会导致整个 table 重排。

# 强缓存和协商缓存
强缓存：Cache-Control: max-age=xxx（HTTP/1.1，相对时间，优先级更高）
协商缓存：
  服务器响应头：ETag: 资源唯一标识（哈希/版本号）
  浏览器下次请求：If-None-Match: 上次的 ETag
  服务器对比标识：
  一致 → 304
  不一致 → 200 + 新资源
